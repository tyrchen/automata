# Rust 工作流引擎产品规格说明书

## 执行摘要

本文档定义了一个基于 Rust 构建的高性能工作流引擎的完整产品规格。该引擎旨在提供类似 n8n 的工作流自动化能力，但通过 Rust 的性能优势和自定义 DSL 实现更高的执行效率和更好的可扩展性。

### 项目核心价值
- **高性能执行**：利用 Rust 的零成本抽象和内存安全特性
- **简洁的 DSL**：提供易于理解和编写的领域特定语言
- **模块化架构**：支持灵活的节点扩展和自定义集成
- **可视化支持**：自动生成工作流的只读 UI 展示

## 1. 项目背景与目标

### 1.1 项目背景

随着业务自动化需求的增长，工作流引擎成为连接不同系统和自动化业务流程的关键基础设施。现有解决方案如 n8n、Zapier 等虽然功能强大，但在性能、可扩展性和定制化方面存在限制。

### 1.2 项目目标

- **性能目标**：实现毫秒级的节点执行延迟，支持每秒处理 10,000+ 工作流实例
- **可扩展性**：提供简单的节点开发接口，支持社区贡献
- **易用性**：通过直观的 DSL 降低使用门槛
- **可靠性**：提供工作流执行的事务性保证和错误恢复机制

## 2. 竞品分析

### 2.1 n8n

**架构特点**：
- 基于 Node.js 的事件驱动架构
- 使用 JSON 格式存储工作流定义
- 提供可视化的工作流编辑器
- 支持 200+ 预建集成

**核心组件**：
```typescript

  "nodes": [{
    "id": "node1",
    "type": "n8n-nodes-base.httpRequest",
    "position": [250, 300],
    "parameters": {
      "url": "https://api.example.com/data",
      "method": "GET"
    }
  }],
  "connections": {
    "node1": {
      "main": [["node2"]]
    }
  }
}
```

**数据传递机制**：
- 节点间通过 JSON 对象传递数据
- 支持数据转换和映射
- 使用表达式语言进行动态值计算

### 2.2 Zapier

**特点**：
- SaaS 模式，完全托管
- 强大的应用生态系统（5000+ 集成）
- 简化的线性工作流模型
- 基于触发器-动作的设计模式

**局限性**：
- 闭源系统，无法自托管
- 定价模式对高频使用场景不友好
- 有限的自定义能力

### 2.3 Make (原 Integromat)

**优势**：
- 可视化的数据映射
- 支持复杂的分支和循环逻辑
- 强大的数据转换能力

**架构特点**：
- 基于场景（Scenario）的概念
- 模块化的操作设计
- 实时和计划执行模式

### 2.4 Windmill

**创新点**：
- 使用 TypeScript/Python 脚本定义逻辑
- Git 集成的版本控制
- 支持复杂的审批流程
- 提供 API 优先的设计

**技术栈**：
- Rust 后端（类似我们的选择）
- PostgreSQL 存储
- React 前端

#### 5.2.1 DSL 解析器

```rust
pub struct DslParser {
    schema_validator: SchemaValidator,
    expression_parser: ExpressionParser,
}

impl DslParser {
    pub async fn parse(&self, input: &str) -> Result<WorkflowDefinition, ParseError> {
        // 1. YAML 解析
        let yaml_value: Value = serde_yaml::from_str(input)?;

        // 2. 模式验证
        self.schema_validator.validate(&yaml_value)?;

        // 3. 构建 DAG
        let dag = self.build_dag(&yaml_value)?;

        // 4. 验证 DAG (检查循环等)
        self.validate_dag(&dag)?;

        // 5. 解析表达式
        let expressions = self.parse_expressions(&yaml_value)?;

        Ok(WorkflowDefinition {
            metadata: self.parse_metadata(&yaml_value)?,
            triggers: self.parse_triggers(&yaml_value)?,
            dag,
            expressions,
            tests: self.parse_tests(&yaml_value)?,
        })
    }

    fn build_dag(&self, value: &Value) -> Result<DirectedGraph, ParseError> {
        let mut graph = DirectedGraph::new();

        // 解析节点
        let nodes = value.get("nodes").ok_or(ParseError::MissingNodes)?;
        for (node_id, node_def) in nodes.as_mapping()? {
            let node = self.parse_node(node_id, node_def)?;
            graph.add_node(node);
        }

        // 解析连接
        let connections = value.get("connections").ok_or(ParseError::MissingConnections)?;
        for conn in connections.as_sequence()? {
            let edge = self.parse_connection(conn)?;
            graph.add_edge(edge)?;
        }

        Ok(graph)
    }
}
```

#### 5.2.2 执行引擎

```rust
pub struct ExecutionEngine {
    node_registry: Arc<NodeRegistry>,
    state_manager: Arc<StateManager>,
    scheduler: Arc<TaskScheduler>,
}

impl ExecutionEngine {
    pub async fn execute_workflow(
        &self,
        workflow_id: Uuid,
        trigger_data: Value,
    ) -> Result<ExecutionResult, ExecutionError> {
        // 1. 创建执行上下文
        let mut context = ExecutionContext::new(workflow_id, trigger_data);

        // 2. 加载工作流定义
        let workflow = self.state_manager.load_workflow(workflow_id).await?;

        // 3. 初始化执行状态
        let execution_id = Uuid::new_v4();
        self.state_manager.init_execution(execution_id, &workflow).await?;

        // 4. 创建执行计划
        let plan = self.create_execution_plan(&workflow, &context)?;

        // 5. 执行节点
        for stage in plan.stages {
            let futures: Vec<_> = stage.nodes.into_iter()
                .map(|node_id| self.execute_node(node_id, &mut context))
                .collect();

            // 并行执行同一阶段的节点
            let results = futures::future::join_all(futures).await;

            // 检查错误
            for result in results {
                if let Err(e) = result {
                    return self.handle_execution_error(execution_id, e).await;
                }
            }

            // 更新状态
            self.state_manager.update_stage_complete(execution_id, stage.id).await?;
        }

        Ok(ExecutionResult {
            execution_id,
            status: ExecutionStatus::Completed,
            outputs: context.collect_outputs(),
        })
    }

    async fn execute_node(
        &self,
        node_id: NodeId,
        context: &mut ExecutionContext,
    ) -> Result<(), ExecutionError> {
        // 获取节点实例
        let node = self.node_registry.get_node(&node_id.node_type)?;

        // 准备输入
        let input = self.prepare_node_input(&node_id, context).await?;

        // 执行节点
        let output = node.execute(context, input).await?;

        // 保存输出
        context.set_node_output(node_id, output);

        Ok(())
    }
}
```

        self.register("conditional", Arc::new(ConditionalNode::new()));
        self.register("foreach", Arc::new(ForEachNode::new()));
        // ... 更多内置节点
    }

    pub fn register(&self, node_type: &str, node: Arc<dyn Node>) {
        let mut nodes = self.nodes.write().unwrap();
        nodes.insert(node_type.to_string(), node);
    }

    pub async fn load_custom_nodes(&self, path: &Path) -> Result<(), LoadError> {
        for loader in &self.loaders {
            if loader.can_load(path) {
                let nodes = loader.load(path).await?;
                for (node_type, node) in nodes {
                    self.register(&node_type, node);
                }
            }
        }
        Ok(())
    }
}

#[async_trait]
impl Node for HttpNode {
    fn node_type(&self) -> &'static str {
        "http"
    }

    async fn execute(
        &self,
        context: &mut ExecutionContext,
        input: NodeInput,
    ) -> Result<NodeOutput, ExecutionError> {
        let config = input.config;
        let url = context.evaluate_expression(&config["url"])?;
        let method = config["method"].as_str().unwrap_or("GET");

        let response = match method {
            "GET" => self.client.get(url).send().await?,
            "POST" => {
                let body = context.evaluate_expression(&config["body"])?;
                self.client.post(url).json(&body).send().await?
            }
            _ => return Err(ExecutionError::UnsupportedMethod(method.to_string())),
        };

        let status = response.status();
        let body = response.json::<Value>().await?;

        Ok(NodeOutput {
            data: json!({
                "status": status.as_u16(),
                "body": body,
            }),
            metadata: Default::default(),

```rust
pub struct StateManager {
    db: PgPool,
    cache: Arc<Mutex<CacheLayer>>,
}

impl StateManager {
    pub async fn init_execution(
        &self,
        execution_id: Uuid,
        workflow: &WorkflowDefinition,
    ) -> Result<(), Error> {
        // 在数据库中创建执行记录
        sqlx::query!(
            r#"
            INSERT INTO executions (id, workflow_id, status, started_at)
            VALUES ($1, $2, $3, $4)
            "#,
            execution_id,
            workflow.id,
            ExecutionStatus::Running as i32,
            Utc::now(),
        )
        .execute(&self.db)
        .await?;

        // 初始化缓存状态
        let initial_state = ExecutionState {
            execution_id,
            workflow_id: workflow.id,
            current_stage: 0,
            node_states: HashMap::new(),
            global_context: json!({}),
        };

        self.cache
            .lock()
            .await
            .cache_workflow_state(&execution_id.to_string(), &initial_state)
            .await?;

        Ok(())
    }

    pub async fn checkpoint_node_state(
        &self,
        execution_id: Uuid,
        node_id: &str,
        state: &NodeState,
    ) -> Result<(), Error> {
        // 更新数据库
        sqlx::query!(
            r#"
            INSERT INTO node_executions (execution_id, node_id, status, output, executed_at)
            VALUES ($1, $2, $3, $4, $5)
            ON CONFLICT (execution_id, node_id)
            DO UPDATE SET status = $3, output = $4, executed_at = $5
            "#,
            execution_id,
            node_id,
            state.status as i32,
            serde_json::to_value(&state.output)?,
            state.executed_at,
        )
        .execute(&self.db)
        .await?;

        // 更新缓存
        let mut cache = self.cache.lock().await;
        let cache_key = format!("execution:{}:node:{}", execution_id, node_id);
        cache.redis.setex(cache_key, serde_json::to_string(state)?, 3600).await?;

        Ok(())
    }
}

    pub target: String,
    pub target_port: String,
    pub label: Option<String>,
    pub condition: Option<String>,
}

// 转换器实现
impl WorkflowDefinition {
    pub fn to_visual(&self) -> VisualWorkflow {
        let layout = self.calculate_layout();

        let nodes = self.dag.nodes()
            .map(|node| VisualNode {
                id: node.id.clone(),
                node_type: node.node_type.clone(),
                label: node.config.get("label")
                    .and_then(|v| v.as_str())
                    .unwrap_or(&node.id)
                    .to_string(),
                position: layout.get_position(&node.id),
                ports: self.calculate_ports(node),
                status: None,
                metadata: node.config.clone(),
            })
            .collect();

        let edges = self.dag.edges()
            .map(|edge| VisualEdge {
                id: format!("{}-{}", edge.source, edge.target),
                source: edge.source.clone(),
                source_port: "output".to_string(),
                target: edge.target.clone(),
                target_port: "input".to_string(),
                label: edge.label.clone(),
                condition: edge.condition.clone(),
            })
            .collect();

        VisualWorkflow {
            nodes,
            edges,
            layout: layout.info,
        }
    }

    fn calculate_layout(&self) -> Layout {
        // 使用 Sugiyama 算法进行分层布局
        let mut layout = SugiyamaLayout::new();
        layout.calculate(&self.dag)
    }
}
```

async fn handle_execution_stream(
    mut socket: WebSocket,
    execution_id: Uuid,
    state: Arc<AppState>,
) {
    let mut rx = state.execution_manager.subscribe(execution_id).await;

    while let Ok(update) = rx.recv().await {
        let message = serde_json::to_string(&update).unwrap();

        }
    }
